Hugo+GithubPagesなブログをmacで構築してみた

思い立った

昔作ったgithubのアカウントを掘り出したのをきっかけに、
環境構築手順でもブログ形式で残しておきたいなあと思い立った私はブログを作成したんです(錯乱

なるべく楽して作りたい

面倒くさがりなので(なるべく)1クリックで作りたい、出来ればgithubで記事を管理したい。。。。
そんな思いから「github　ブログ」と安直にぐぐったところ選択肢が幾つか出て来た。

  • Jekyll
  • hugo

この二つが目についたので調べてみる。
そしてhugoはGO言語で速くて構築が簡単でマークダウン言語でさくさく記事が書けて素晴らしいよ！！(曲解)
を信じてぼちぼち導入することにしました。

前提として

git部分はあまり詳しく説明しないので、この程度は出来る方向けです。

  • githubアカウントを持ってる
  • リポジトリ作成、コミット、プッシュ程度は出来る
hugoのインストール

早速インストールコマンドをぶち込む。

$brew install hugo

(´･ω･`)ひゅー！！めっちゃらくう！！！ぶりゅーさまさまだぜ！！！
ちゃんとインストール出来たかをチェック

$hugo version
Hugo Static Site Generator v0.14 BuildDate: 2015-06-17T05:43:06+09:00

いいいいいいいいいえす素晴らしい。
何故インストールってのはこんなにも胸ときめくのか。

新しいブログを作ろう

適当なディレクトリに移動して、hugoブログを新規作成する。
私は ~/work 配下を作業領域としているのでそこで実行してみた。

$hugo new site <site name>

これで ディレクトリが作成された筈。
早速潜って内容を確認する。

$cd <site name>
$ls -l
# archetypes、config.toml、content、data、layouts、static が入っている筈
テーマが割と揃ってる

hugoはテーマを指定しなければindex.html(後述のコマンドで生成する)が真っ白になるので必ず指定してあげる。
勿論webデザインに自信があるなら自作しても良いのだけれど、hugoはそこそこテーマが揃っているのでまずはそこから選んでみる。

  1. ここにテーマカタログがあるので好みのものを選ぼう。
  2. 私は割と下の方にあるmaterial-designを選択。このブログもこのデザイン。
  3. テーマのリポジトリページに遷移するので右っかわにある HTTPS clone URL てのをコピーする。
  4. ルートディレクトリ直下に themes ディレクトリを作りその中で上記リポジトリをcloneする。

実際には以下の通りコマンドを投入してみた。

$cd <site name>
$mkdir themes
$cd themes
$git clone <コピーしたHTTPS clone URL>

themesディレクトリ配下に選んだテーマ名のディレクトリが作成されていたら成功だ。
このthemesディレクトリにはいくらでもテーマを保管しておいて構わないし、テーマの変更もコマンド一行で済んでしまうので気になるものや迷っているものがあったら入れておいてもいいと思う。

一瞬でブログのプレビューが出来る

恐らくhugoの便利なところの一つだけれど、コマンド一つで一瞬でブログを確認することが出来る。
勿論ローカルで。

ディレクトリ配下で以下のコマンドを実行してみる。

$hugo server -t <さっき選んだテーマ名>
0 of 2 drafts rendered
0 future content 
0 pages created
0 paginator pages created
0 tags created
0 categories created
in 30 ms
Serving pages from <site name>/public
Web Server is available at http://127.0.0.1:1313/
Press Ctrl+C to stop

こんな感じの結果が出ると思う。
こいつが言ってることはつまり、、、、

(´･ω･`)指定されたテーマが適用された public ディレクトリを作ってやったよ！！

(´･ω･`)ついでにローカルホストの1313ってとこで確認出来るようにしてやったから見てみな！！！！

ってことなんだ。
早速 http://127.0.0.1:1313/ に、レッツアクセス。
勿論 http://localhost:1313/ このURLからでも同じだ。

テーマカタログで選んだテーマがそのまま、(記事は0件だが)表示されているはず。

先にも話したが、ここでテーマを指定していないと(つまり hugo server のみでコマンドを叩くと)
public/index.htmlが空で生成されてしまうので注意だ。
(※つまり私がかなりハマったってことなんだ。。。。。)

記事を作る

いよいよ記事を作るのでここでもhugoコマンドのお世話になる。
本当至れり尽くせりだなあと思うんだ。

$hugo new post/first.md

を実行してみると、空っぽの筈だったcontent/配下に post/first.md が生成される。
ブログ記事は今後ここに溜まっていくことになる。
このファイルに記述するのはマークダウン言語ってものなんだけど、それがなんぞやとか記述方法とかは別途調べてほしいな。
